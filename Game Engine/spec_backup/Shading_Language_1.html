Null Engine Shading Language Version 1

Format

A Null Engine Shading Language file consists of a series of blocks and allows arbitrary text outside of blocks that is ignored.
Blocks are in the format:

$BLOCK_NAME
block body
$END

Block names and the END keyword are not case sensitive.

Include

Null Engine Shading Language extends GLSL with the capability to include code with the syntax:

#include "file in global file path"
#include "BLOCK_NAME:file in global file path"
#include <file in current folder>
#include <BLOCK_NAME:file in current folder>

The include options that specify a block name will include only the block that is specified, where as include options that don't specify a block will include the entire file.

Special Blocks

VS

The VS block represents the GLSL vertex shader and expects its contents to be valid GLSL code (extended with includes)

FS

The FS block represents the GLSL fragment shader and expects its contents to be valid GLSL code (extended with includes)

GS

The GS block represents the GLSL geometry shader and expects its contents to be valid GLSL code (extended with includes)

VERSION

The VERSION block is used to include the GLSL version. It expects the format to be:

version_number profile

version_number represents the GLSL version, valid values are: 110, 120, 130, 140, 150, 300, 310, 330, 400, 410, 420, 430, 440, 450
profile represents the GLSL profile

  • For versions less than 150 profile is not supported
  • For versions 300 and 310 the profile is required and must be es
  • For version 150, 330, 400, 410, 420, 430, 440, 450 profile is optional and can be core or compatibility, if no profile is specified the default is core

Required Blocks

The VS, FS and VERSION blocks are required for the shader to be compiled.
